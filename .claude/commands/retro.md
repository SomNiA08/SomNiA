# /retro — 회고 · 개정 · 역이식

사이클을 닫는 유일한 문이다. 회고 → **사람 승인(사이클당 1회)** → 프로젝트·원본 킷 동시 반영.

**원본 킷 위치: 형제 디렉토리 중 README.md 첫 줄이 `# harness-kit`인 폴더** (드라이브·폴더명은 컴퓨터마다 달라도 된다 — 이름이 아니라 내용으로 찾는다. 이 프로젝트 자체가 원본 킷이면 자기 자신이 곧 그 폴더다).
**일치가 2개 이상이면 중단하고 사람에게 보고한다** (아래 금지 절 — 첫 일치 채택 금지).

## 절차

1. `state/state.json` 확인 — `status`가 `report_done`이 아니면 안내 후 중단.
1-1. **사건 장부 집계 (자동 회고 재료 · HANDOFF W4)**: `state/incidents.jsonl`이 있으면 읽어,
   이번 사이클에 훅이 남긴 사건(pre-tool-use 차단 · post-tool-use 경고 · stop-guard 포기)을
   유형·빈도로 집계한다. 이 집계를 retrospector 에게 넘겨 **실패 후보의 1순위**로 삼는다
   (사람 기억이 아니라 기록이 회고의 시작점이다). 장부가 없거나 비었으면 이 단계는 건너뛴다.
2. `retrospector` 에이전트 실행 → `retro/<날짜>-retro.md` 작성.
   구조: 실패 → 어느 규칙의 구멍 → 재발 방지 금지문 초안 (증거 경로 필수, retrospector 규칙).
3. **개정안 승인 (사이클에서 유일한 사람 개입 지점)**:
   - 개정안이 있으면 사용자에게 항목별로 제시하고 두 가지를 받는다 (AskUserQuestion 활용):
     ① 승인/기각 · ② 적용 범위 = **킷 공통**(모든 프로젝트에 유효한 벽) / **프로젝트 전용**(이 도메인에만 유효).
   - 범위가 애매하면 기본값은 **프로젝트 전용** — 킷 오염보다 킷 누락이 낫다. 다음 프로젝트에서
     같은 구멍이 또 나오면 그때 공통으로 승격한다.
   - 승인된 개정안마다 **한 가지를 더 판정한다**: "이 벽을 **코드·테스트로 승격**할 수 있는가?"
     (훅 패턴 · 게이트 스크립트 · 회귀 테스트). 전용 구현이라도 **추출할 공통 패턴**이 있으면 병기한다.
     산문 벽은 약한 모델이 못 지킬 수 있지만 기계 벽은 지킨다 — 승격 가능하면 개정문에
     `[승격 후보: <방법>]`을 남긴다. (HANDOFF W4 · F2 재발 방지)
   - 개정안이 없으면 이 단계 생략.
4. **승인된 개정안 반영 (반자동 역이식)** — 항목마다:
   - (a) 이 프로젝트의 해당 문서(SOUL/AGENTS/CLAUDE.md)에 Edit로 append — 공통·전용 모두
   - (b) **킷 공통 판정 항목만** 원본 킷(형제 디렉토리 중 README `# harness-kit` 폴더)의 같은 문서에 Edit로 append
     (현재 프로젝트가 원본 킷 자체면 (a)와 동일하므로 생략)
   - (c) 반영한 곳의 `CHANGELOG.md`에 이력 append: 날짜 · 출처 프로젝트 · 범위 · 개정 내용 · 근거(회고 경로)
   - 기각된 개정안은 회고 파일에 "기각 + 사유"로 남긴다 (Edit append).
5. **원본 킷 커밋·푸시 (git 이력 = 자동 백업)** — 역이식이 1건이라도 있으면 즉시:
   - 원본 킷에서 `git add -A` → `git commit`(메시지: `retro: <프로젝트> cycle N — <개정 한 줄 요약>`)
   - `git push origin main` (일반 푸시만) → 푸시 성공을 출력으로 확인.
   - **1 개정 = 1 커밋** — 개정 전 버전은 git 이력에 전부 남는다. 잘못된 개정의 복구는
     `git revert <커밋>` 한 번이면 된다 (이력이 곧 백업이므로 별도 백업 불필요).
6. 마무리:
   - `state/state.json`: `retro_path` 기록, `status: "retro_done"`
   - `state/.retro-attempts` sentinel 삭제 (Stop 훅 가드 해제)
   - `state/incidents.jsonl`: 1-1의 집계를 회고 파일에 남겼으면 장부를 비운다(빈 파일로 덮어쓰기) — 다음 사이클용.
   - `log.md` 한 줄 append: `- <날짜> cycle N 회고 완료 · 개정 <승인수>/<제안수> (공통 <n>·전용 <n>)`
7. 종료 보고: 회고 경로 · 승인/기각·범위 집계 · 역이식 반영 파일 · 킷 커밋 해시만.

## 금지

- ⛔ 사람 승인 없이 규칙 문서를 개정하지 마라. retrospector의 개정안은 항상 *제안*이다 (AGENTS §1).
- ⛔ **프로젝트 전용 개정을 킷에 역이식하지 마라.** 승인 때 받은 범위 분류를 따른다. 도메인 규칙이
  킷에 새는 순간 킷은 범용성을 잃는다.
- ⛔ **킷 공통 판정 항목의 역이식을 건너뛰지 마라.** 건너뛰면 킷이 성장하지 않는다.
- ⛔ **원본 킷을 수정해 놓고 커밋 없이 끝내지 마라.** 개정과 커밋 사이의 틈이 백업 구멍이다.
- ⛔ **원본 킷에 force-push(`--force`) 하지 마라.** 이력이 곧 백업이다 — 되돌리기는 `git revert`로만.
- ⛔ 개정문을 요약·의역해 옮기지 마라. 승인된 문구 그대로, 반영처에 동일하게 append.
- ⛔ 회고 없이 개정만 하지 마라. 개정안의 출처는 항상 회고 파일이어야 한다.
- ⛔ **형제 탐색에서 README 첫 줄 `# harness-kit` 일치가 2개 이상이면 첫 일치를 채택하지 마라.** 중단하고
  일치한 폴더 목록을 사람에게 보고한다 — tie-break 없는 첫 일치 채택은 역이식 오착지의 기본값이다
  (2026-07-10 실측: 킷 사본 README를 가진 리포를 `fleet-sync.ps1`이 킷으로 오인). 사본 README를 가진
  리포가 자기 명명으로 교체돼 일치가 1개가 된 뒤에만 재개한다. (v0.36 — 출처: 2026-07-10 함대 검수 치명 1)
- ⛔ **[승격 후보]가 있는 개정안을 산문 벽만 채택하고 승격 후보를 미이행으로 닫지 마라.** 승격 후보를 이번에
  이행하지 않기로 하면, 그 사유와 "다음 재발 시 즉시 이행"을 명시해 다음 회고가 추적하게 한다. 산문 벽 단독
  채택은 결함이 아니라 트리거 하나만 막는다(cycle 2 개정안 1 → cycle 4 실패 A 재발이 그 실증).
  [승격 후보: 회고 개정안 총괄표에 '승격 이행상태(이행/유예+사유)' 열 상시화 + 절차 2 반복 점검이 직전 사이클
  총괄표의 미이행 승격 후보를 자동 대조 항목으로 포함] (v0.35 — 출처: ai-worklog cycle 4, retro/2026-07-08-retro.md 메타관측)
- ⛔ **사건 장부가 비었거나 훅 deny만 있다는 이유로 "실패 없음"으로 회고를 닫지 마라.** 장부는 훅이
  본 층만 기록한다 — 에이전트 산출물의 규약 위반, 게이트 트리거 범위 밖 결함, 커맨드 스펙 누락,
  로드된 절차의 출처 오염은 장부에 남지 않는다. retrospector 브리프에는 장부와 **함께** 이번 사이클
  각 걸음의 (a) 에이전트 반환 메시지 원문 (b) REVISE 발동/미발동 이력 (c) 판정 산출물이 "공백·측정
  불가·약한 PASS"로 표기한 칸 목록을 반드시 넘긴다 (2026-07-12 cycle 1: 실패 4건 중 장부 적재 0건).
  [승격 후보: state/step-notes.jsonl(mutable 계층) 신설 — 메인 루프가 걸음마다 {step, agent,
  returned_summary, gate_result, revise_count}를 append하고, /retro 절차 1-1이 incidents.jsonl과
  함께 소비한다. 훅으로 강제할 수 없는 층이므로 산문 벽 + 커맨드 절차 개정이 주력이고, 기계 승격분은
  "각 커맨드 종료 절에 step-notes append를 절차 항목으로 못박고, /retro 절차 1-1이 step-notes 부재 시
  경고를 출력"까지가 한계] (v0.37 — 출처: somnia-hub cycle 1, retro/2026-07-12-retro.md 실패 6)
