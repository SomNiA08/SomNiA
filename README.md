# harness-kit — 회의형 하네스 킷 (한글 우선판)

강사님의 하네스 정교화 원칙 7가지를 반영한 **재사용 가능한 규칙 문서 + 에이전트 세트**다.
기본 흐름: **안건·자료 로드 → 에이전트 회의(발언 원본 기록) → 서기 레포트 → 회고**.
외부 연동(강의에서 예시로 든 Notion·Slack 등)은 기본 포함이 아니다 — 필요해질 때
CLAUDE.md의 발행처 등록 표에 등록한 뒤에만 쓴다. 미등록 상태에서는 전부 로컬 파일로 동작한다.

## 파일 구조

```
harness-kit/
├── SOUL.md                     ← 헌법 (불변 · 최상위 금지)
├── AGENTS.md                   ← 에이전트 공통 규칙 + 회의 파이프라인
├── CLAUDE.md                   ← Claude Code 배선 (MCP·훅·권한·크론·발행처 등록 표)
├── CHANGELOG.md                ← 규칙 개정 이력 (역이식 착지점, append-only)
├── log.md                      ← 걸음 기록 (append-only)
├── README.md                   ← 이 파일 (원칙 매핑 + 사용법)
├── state/ rounds/ wiki/ report/ retro/   ← 상태·발언원본(불변)·정리·레포트·회고
└── .claude/
    ├── settings.json           ← 훅 배선
    ├── hooks/                  ← 벽 4종 (.mjs, 2026-07-03 실행 검증 완료)
    │   ├── session-start.mjs       벽1: 헌법+상태 재주입
    │   ├── pre-tool-use.mjs        벽2: append-only + rounds/ 불변
    │   ├── post-tool-use.mjs       벽3: AI-slop·증거부실 경고
    │   └── stop-retro-guard.mjs    벽4: 회고 없는 종료 차단 (3회 상한)
    ├── commands/
    │   ├── meeting.md          ← /meeting <안건> — 사이클 시작
    │   ├── meeting-round.md    ← /meeting-round — 라운드 1회 (/loop 호환)
    │   └── retro.md            ← /retro — 회고·승인·역이식 (사이클을 닫는 유일한 문)
    └── agents/
        ├── moderator.md            ← 의장 (안건·발언권·집계)
        ├── panelist-pm.md          ← 기획 토론자 (새 토론자 추가 시 템플릿)

        ├── scribe.md               ← 서기 (유일한 외부 발행자)
        └── retrospector.md         ← 회고 담당 (반성문 → 규칙 개정 제안)
```

## 규칙이 쌓이는 구조 (반자동 역이식)

```
프로젝트에서 사이클 종료 → Stop 훅이 회고 없는 종료를 차단 → /retro
  → retrospector 개정안 → 사람 승인 1회 + 범위 분류(킷 공통/프로젝트 전용)
  → 공통만 원본 킷(C:\work\harness-kit)에 역이식 + CHANGELOG 기록
  → 원본 킷 git 커밋·푸시 (1 개정 = 1 커밋 = 자동 백업)
```

완전 자동이 아니라 **구조로 강제되는 반자동**이다 — 규칙 개정은 사람이 재판관이고(승인 1회),
"역이식을 깜빡함"은 커맨드·훅 구조가 차단한다. 킷은 프로젝트를 거칠 때마다 강해진다.

**원본 유실 방어 3중벽**: ① 덮어쓰기는 pre-tool-use 훅이 차단(append만 가능) ②
개정 전 버전은 git 이력에 전부 보존 — 잘못된 개정은 `git revert` 한 번으로 복구, force-push 금지 ③
프로젝트 전용 규칙은 범위 분류에서 걸러져 킷에 들어오지 않는다 (애매하면 전용이 기본값).

## 강사님 작성 철학 → 어떻게 지켰나

- **"해야 하는 것(30% 미만) <<<<< 하면 안 되는 것(NEVER/DON'T)"**
  → 모든 규칙 문서·에이전트 파일이 금지문 중심이고, DO는 문서당 3항목 이내의 "최소" 절로만 존재한다.
- **"하네스를 깎아가면서, 기강을 잡아가면서, 반성문/회고록을 스스로 복기해 작성하게"**
  → `retrospector` 에이전트 + SOUL §6 (회고 없는 종료 금지) + CLAUDE §4 (stop 훅 회고 가드).
  회고는 반드시 "실패 → 규칙 구멍 → 금지문 개정안"으로 끝나야 하고, 그 개정안이 하네스를 깎는다.
- **"md는 가급적 영어로 (토큰 절약)"**
  → 지금은 이해·수정이 쉬운 한글 우선판. 규칙이 몇 사이클 돌며 안정되면 영어로 압축 전환한다
  (SOUL.md 작성 규약에 명시). 전환 시에도 500줄 미만·금지 70% 규칙은 그대로 유지.

## 사용법 (시작 전 체크리스트)

1. 이 폴더를 새 프로젝트 폴더(`C:\work\<영문이름>\`)로 복사한다.
2. (선택) 외부 연동이 필요하면 **CLAUDE.md 발행처 등록 표의 빈칸(⬜)**을 채운다.
   **빈칸이 남아 있으면 그 대상으로는 발행이 금지되고, 산출물은 로컬 파일로만 남는다.**
3. (선택) 등록한 커넥터의 MCP를 연결한다 (`claude mcp add ...` 또는 커넥터 설정).
4. 필요한 관점만큼 `panelist-*.md`를 추가한다 (panelist-pm.md를 템플릿으로 복사,
   AGENTS.md 역할 표에 등록).
5. 커맨드 3종·훅 4종은 실물로 포함돼 있다 (2026-07-03 실행 검증 10케이스 통과).
   크론(정기 회의·회고 예약)만 프로젝트별로 설정한다 — CLAUDE.md §6.
