# TOP-WALLS — 재주입용 상위 벽 10 (전문은 SOUL·CLAUDE·AGENTS)

세션 시작·압축 때 session-start 훅이 이 목록을 SOUL 직후 재주입한다. 약한 모델은 긴 규칙 전문보다
짧고 강한 벽 목록에 훨씬 잘 붙는다 — 이 10개는 "잊으면 사고 나는" 최상위 벽이다. 전문은 각 문서 참조.

1. 증거 없이 "됐다" 금지 — 명령 출력·경로·해시로만 증명한다. (SOUL §1)
2. 존재 ≠ 정확 ≠ 작동 — "만들었다"를 "작동한다"로 바꿔 말하지 마라. (SOUL §1)
3. 소스 계층(SOUL/AGENTS/CLAUDE/log/CHANGELOG·원본 기록 디렉토리) Write 덮어쓰기 금지 — 추가는 Edit. (SOUL §2)
4. 걸음의 원자 단위 = 산출 + 기록 + 커밋. 미커밋 걸음은 미완이다. (CLAUDE §2-1)
5. add와 commit 분리 실행 — && 묶기 금지, 커밋 증명은 직후 git log 해시. (CLAUDE §2-1)
6. 세션 시작 시 git status로 미커밋 잔재 확인 — 발견 시 새 걸음 전에 복구 커밋. (CLAUDE §2-1)
7. 에이전트 배분 전 레지스트리 등록 확인 — 등록은 세션 속성, 매 세션 새로 본다. (CLAUDE §3)
8. 회고 없이 사이클 종료 금지 — done = 레포트 + 회고. (SOUL §6)
9. 한 커맨드 호출 = 한 걸음 — 여러 단계 몰아 하기 금지. (SOUL §4)
10. 자기 산출물 자기 평가 금지 — 평가는 격리된 devil·critic·retrospector에게. (SOUL §5)
