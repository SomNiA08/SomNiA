# model-gym — 새 모델 캘리브레이션 체육관 (박제 2026-07-06 · 정답본 = Fable 5 직접 작성)

새 모델을 함대에 앉히기 전에 **역할군 합격 여부를 측정**하는 골든 과제 3종이다.
MODELS.md의 벽 — "⛔ 새 모델을 검증 없이 판정 역할군에 앉히지 마라" — 를 실행하는 도구다.

## 과제 ↔ 역할군 대응

| 과제 | 무엇을 측정하나 | 합격 시 맡길 수 있는 역할군 (MODELS.md) |
|---|---|---|
| `task-a-meeting/` | 술어화·토론·악마(마지막 발언·3종 세트)·집계 | **판정** (devils-advocate·critic류) + 생산-내부 (panelist·moderator) |
| `task-b-content/` | record 사실 범위 준수·훅·시행착오 서사·규격 | **생산-외부발행** (writer류) |
| `task-c-doc/` | 근거 1:1 인용·[미확인] 규율·층위 통제 | **생산-외부발행** (doc-writer) + 판정 보조 (doc-critic 대조 능력) |

## 표준 (2026-07-10 사용자 확정 — 하이브리드 · 이 절이 유일한 표준이다)

이 문서 안에 측정 방식이 두 갈래로 병존해(사용법 1 "격리 폴더 사람 세션" vs provenance 절 "킷 세션 Agent
호출") 각각 유일한 표준을 자처했고, `audit.ps1` [5]는 후자(agentId)를 요구해 전자와 충돌했다. 확정:

| 단계 | 어디서 · 누가 | 왜 |
|---|---|---|
| **측정** | **격리 빈 폴더에서 사람이 새 모델 세션을 연다** (사용법 1) | 정답본 열람을 **구조로** 차단한다. 미검증 새 모델에게 "정답 보지 마라" 프롬프트 순종을 전제하는 것은 캘리브레이션의 목적과 순환이다 (2026-07-06 mandela 검수가 닫은 누수) |
| **채점** | **킷 세션에서 Agent 호출** (측정과 **다른 모델**) | 채점은 정답본 열람이 직무라 누수 개념이 없고, `agent-ledger.mjs` 훅이 채점 모델·agentId를 기계 기록한다 |
| **판정·MODELS 확정** | 사람 | 시험 통과는 바닥선 확인이지 승격 근거가 아니다 |

매니페스트에는 **측정 = 사람 실행(시험 폴더 경로·실행 날짜)**, **채점 = agentId**를 나눠 적는다.
`audit.ps1` [5]는 이 형식(`측정 실행: 사람`)을 정식으로 허용한다 — 측정 행의 agentId 부재는 결함이 아니다.
(근거: `somnia-hub/검수-개선-설명서-2026-07-10.md` §3-D · FLEET-KITS P0-6 하이브리드 권고안. 사용자 확정 2026-07-10.)

⛔ **측정을 킷 세션 Agent 호출로 하지 마라.** provenance가 완벽해져도 정답본과 같은 디스크에 앉는다.
⛔ **채점을 사람만으로 끝내고 매니페스트를 생략하지 마라.** 사람 채점은 허용되지만(사용법 2), 그때도
   매니페스트에 `채점 실행: 사람`을 적고 [5]가 그 형식을 본다.

## 사용법 (새 모델 첫날 — 캘리브레이션 사이클)

1. **빈 폴더에 input.md 3개만 복사하고, 측정 세션은 반드시 거기서 연다.** 과제 지시문도 input.md 안에 있다.
   이 킷 폴더에서 열면 안 되는 이유: 정답본이 같은 폴더 디스크에 있고 **훅은 쓰기만 막지 읽기는 못 막는다**
   — 측정 대상 모델이 정답을 열람하는 순간 측정 무효다. (킷 폴더에서 실행할 준비 명령:)
   ```powershell
   mkdir ..\model-exam
   Copy-Item model-gym\task-a-meeting\input.md ..\model-exam\input-a.md
   Copy-Item model-gym\task-b-content\input.md ..\model-exam\input-b.md
   Copy-Item model-gym\task-c-doc\input.md ..\model-exam\input-c.md
   ```
   ⛔ answer.md(정답본)·scoring.md(채점표)를 새 모델에게 어떤 경로로도 노출하지 마라 — 암기 오염으로 측정이 무효가 된다.
   (참고: 이 킷은 공개 리포라 정답본이 인터넷에도 있다 — 측정 결과가 정답본과 표현까지 비슷하면 암기를 의심하라.)
2. 산출물을 받아 **scoring.md 항목별로 통과/위반 표**를 만든다. 채점은 사람이 하거나,
   **측정 대상과 다른 모델**에 answer.md·scoring.md를 주고 시킨다 — 같은 모델의 다른 세션은 별도로 안 친다
   (같은 모델이 같은 버릇을 같은 잣대로 봐주는 것을 차단).
   ⛔ 측정 대상 모델이 자기 산출물을 자기 채점하는 것 금지 (SOUL §5).
3. 판정: **치명 0건 = 해당 역할군 합격. 치명 1건 이상 = 그 역할군에 그 모델 배치 금지** (권고 위반은 배치 가능하되 기록).
4. 결과대로 `MODELS.md` 현재 값을 갱신한다 (모델 교체 = 표 한 줄).

## provenance 매니페스트 (v0.24 벽의 기계 게이트 — 2026-07-06 신설 · 2026-07-10 하이브리드 정합)

"측정과 채점은 물리적으로 분리돼야 하고 그 분리가 증명돼야 한다"(MODELS.md 캘리브레이션 벽)를 산문이
아니라 기계 기록으로 남긴다. 역할극 산출은 파일 지문이 없지만, **훅이 쓰는 호출 장부에는 실제 호출이
없으면 줄도 없다.**

1. **채점 세션은 반드시 킷 폴더에서 연다** — 킷의 `agent-ledger.mjs` 훅(PostToolUse: Agent)이 채점
   서브에이전트 호출의 모델값·agentId를 `state/agent-calls.jsonl`(로컬 전용)에 자동 기록한다.
   **측정 세션은 격리 폴더에서 사람이 연다**(위 표준 표) — 그 폴더엔 훅이 없으므로 agentId도 없다.
   그것이 결함이 아니라 격리의 대가이며, 매니페스트가 그 사실을 명시한다.
2. 캘리브레이션이 끝나면 **매니페스트를 생성해 커밋**한다: `model-gym/runs/<YYYY-MM-DD>-<모델>.md`
   — 필수 줄: `측정 모델: <값>` · `채점 모델: <값>` · `상태: OK` + 실행 표.
   - 측정 행: `측정 실행: 사람` + 시험 폴더 경로·실행 날짜 (agentId 없음 — 격리 세션이므로 정상)
   - 채점 행: 과제·회차별 `agentId` (장부 `state/agent-calls.jsonl`에서 옮겨 적는다)
3. **scoring.md에 판정 행을 추가하는 커밋에 매니페스트를 함께 커밋한다.** 매니페스트 없는 판정 행 =
   미검증 데이터 — somnia-hub `audit.ps1` 검사 [5]가 주 1회 대조해 WARN을 띄운다
   (매니페스트 부재 · 측정 모델 = 채점 모델 · 채점 agentId 기록 없음).
4. 게이트 신설(2026-07-06) 이전 실측은 `상태: LEGACY`로 소급 불가를 명시한 매니페스트를 둔다 —
   당시 근거는 세션 로그에만 존재함을 숨기지 않는다.

## 벽

- ⛔ **scoring.md 판정 행을 매니페스트 없이 추가하지 마라.** 측정의 provenance(실제 호출 장부)가 없는
  판정은 미검증 데이터다 — 위 매니페스트 절차가 유일한 표준이다. (v0.24 기계 게이트)
- ⛔ **input.md의 픽스처(고정 입력)를 수정하지 마라.** 입력이 바뀌면 모델 간 비교가 무효다.
  과제를 늘리고 싶으면 task-d를 새로 만들지, 기존 입력을 고치지 않는다.
- ⛔ **정답본과 다르다는 이유만으로 감점하지 마라.** 채점 기준은 scoring.md의 치명/권고 항목뿐이다 —
  정답본은 "치명 0의 실물 예시"이지 유일해다.
- ⛔ **픽스처 속 수치·규정을 실제 문서에 재사용하지 마라.** 전부 캘리브레이션용 가짜 데이터다.
- ⛔ 합격 전 모델에게 실전 사이클의 판정(devil·critic·retrospector) 역할을 주지 마라 (MODELS.md 캘리브레이션 벽).
- ⛔ **측정 세션을 정답본이 있는 폴더(이 킷 등)에서 열지 마라.** 위 "표준" 표의 하이브리드가 유일한 절차다.
- ⛔ **측정과 채점을 같은 모델로 하지 마라.** 같은 모델의 다른 세션도 별도로 안 친다 (자기 확인 차단).
