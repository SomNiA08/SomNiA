## 상황별 한 줄 (기본 회의형 기준)

| 상황 | 입력 | 무슨 일이 일어나나 |
|---|---|---|
| 논의할 안건이 생겼다 | `/meeting "안건"` | 의장이 안건을 술어형으로 확정하고 사이클 준비 |
| 회의를 진행하고 싶다 | `/meeting-round` | 토론자 병렬 발언 → 악마의 변호인 마지막 → 집계. 1회 = 1라운드 |
| 라운드를 무인으로 돌리고 싶다 | `/loop /meeting-round` | 합의(또는 최대 라운드)까지 자동 반복 |
| 합의됐고 레포트도 나왔다 | `/retro` | 회고 → 개정안 승인(나) → 원본 킷 반영·백업 |

순서는 항상 같습니다: **`/meeting` → `/meeting-round`(반복) → 레포트 → `/retro`**.
순서를 건너뛰면 하네스가 알아서 막고 다음 할 일을 알려줍니다.

## 커맨드 상세

### `/meeting "<안건>"` — 회의 사이클 시작
- **언제**: 결정이 필요한 안건이 생겼을 때. 이전 사이클이 안 닫혔으면 거부됩니다.
- **무엇**: 안건을 "X 하면 Y 한다" 검사 가능한 형태로 다듬고, 통과 기준을 토론 **전에** 기록.
- **내가 할 일**: 안건 한 줄.

### `/meeting-round` — 라운드 1회
- **무엇**: 토론자들이 병렬로 발언(각자 자기 발언만 기록) → 악마의 변호인이 **항상 마지막**
  발언(낙관으로 끝나는 것 방지) → 의장이 표를 집계. 합의되면 서기가 레포트 작성까지.
- **내가 할 일**: 없음. 발언 원본은 `rounds/`, 레포트는 `report/`에 쌓입니다.

### `/retro` — 회고와 마무리 (사이클을 닫는 유일한 문)
- **무엇**: 회고 작성 → 개정안을 항목별로 나에게 승인 요청(범위: 킷 공통/프로젝트 전용) →
  승인분을 규칙 문서·원본 킷·CHANGELOG에 반영 → 킷 GitHub 자동 백업(1 개정 = 1 커밋).
- **내가 할 일**: 승인/기각 선택 한 번 (사이클 전체에서 유일한 필수 개입).
- **복구**: 잘못 반영된 개정은 `git revert`로 그 건만 되돌릴 수 있습니다 (이력 = 백업).

## 자동으로 되는 것 (훅 — 신경 안 써도 됩니다)

- 세션 시작·재개·컨텍스트 압축 때마다: 헌법(SOUL) + 상태 + 다음 걸음 자동 재주입
- 규칙 문서·발언 원본 덮어쓰기 시도: 자동 차단 (에러가 아니라 벽이 작동한 것)
- 레포트까지 나왔는데 회고 없이 끝내려 하면: 자동으로 `/retro` 요구 (3회까지)
- 산출물에 상투적 표현이 들어가면: 자동 경고

## 새 프로젝트에 이식하는 법 (요약)

1. 이 폴더를 `C:\work\<새이름>\`으로 복사 (`.git` 제외).
2. 에이전트(`.claude/agents/`)와 커맨드(`.claude/commands/`)를 도메인에 맞게 교체.
3. 훅은 `pre-tool-use.mjs` 상단 **상수 3줄**만 조정 (PROTECTED / IMMUTABLE_DIRS / MUTABLE_DIRS).
4. 이 파일(00_사용법)을 새 커맨드 기준으로 고쳐 쓰기.
5. 자세한 설계 원칙과 원칙 매핑은 `README.md`, 규칙은 SOUL → AGENTS → CLAUDE 순서로.

## 막혔을 때

- **지금 어디까지 했더라?** → 그냥 물어보세요. `state/state.json`에 항상 기록돼 있습니다.
- **뭔가 차단됐다는 메시지가 떴다** → 보호벽 작동입니다. 메시지에 이유와 올바른 방법이 나옵니다.
- **커맨드 이름이 기억 안 난다** → 이 파일을 열거나, 세션에서 "사용법 알려줘"라고 하면 됩니다.
